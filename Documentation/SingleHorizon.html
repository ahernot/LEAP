1 Introduction and conventions

This is LEAP Equations This document provides a description of the set of equations in Linopy sEnergy Alternative Planning, a multi-energy planning tool implemented with Linopy and available in open source.

It inherits from xarray . An important thing to understand is that Linopy is built on xarray, which implements vector operations that implicitly perform tensorial products. For example, if one has two xarrays x indexed by [i,k] and y by [j,k], x + y gives an xarray z[i,j,k] where z[i,j,k] = x[i,k] + y[j,k].

Writing conventions Sets (indexes) are in red, Parameters (not optimized and given as input) are in blue and Variables (optimized) in black.

Modeling conventions All power is P, all demand is D (except storage inflow), all cost is C, all unit cost is UC, all energy is E.

Optim conventions All Variables are non negative.

2 Objective function and Demand constraint

min : ∑ nrjopC + ∑ convplC + ∑ stoplC + ∑ flexdplC + exchopP ∗0.01

The last term is used here to avoid a dummy exchange solution. Demand Equation labeled (Ctr_Op_operation_demand) applies on [evo ×at]

tot,hopD = ∑ ctconvopP (Conversion technologies) + ∑ st(stoopPout − stoopPin) (Storage technologies) + ∑ afexchopP −∑ afSwap(at,af)[exchopP] (Exchange between areas) −∑ fdflexdopD (Flexible demand) (1)

where Swap(at,af)[exchopP][at = a,af = b] = exchopP[at = b,af = a]

3 Conversion technology constraints

3.1 Main parameters, variables and constraints

Operation

nrjopC = nrjopUC ∗ tot,hopI[evi ×at] (Ctr_Op_operation_costs)
tot,hopD = exo,hopD[at ×evo ×d] (Ctr_Op_conso_hourly)
tot,hopI = ∑ evo×ct×d&evo∈ctΔTconvopP∕ convopη[evi ×at] (Ctr_Op_operation_costs)
convopP ≤ convplPmax ⁡ ∗ convopAF[evi ×at] (Ctr_Op_capacity)

Planning

convplC = convplUC ∗ convplPmax ⁡ [evi ×at] (Ctr_Pl_capacity)
convplPmax ⁡ ≤ convplPmax ⁡ ,max ⁡ [evi ×at] (Ctr_Pl_max_capacity)
convplPmax ⁡ ≥ convplPmax ⁡ ,min ⁡ [evi ×at] (Ctr_Pl_min_capacity)

3.2 Optional parameters, variables and constraints

∑ dΔTconvopP ≤ convopNHmax ⁡ ∗ convplP[evi ×at] (Ctr_Op_stock)
Diff [convopP; 1h] ≤ convopρmax ⁡ ,1h ∗ convplP[evi ×at] (Ctr_Op_rampPlus)
−Diff [convopP; 1h] ≥ convopρmin ⁡ ,1h ∗ convplP[evi ×at] (Ctr_Op_rampMoins)
Diff [convopP; 2h] ≤ convopρmax ⁡ ,1h ∗ convplP[evi ×at] (Ctr_Op_rampPlus2)
−Diff [convopP; 2h] ≥ convopρmin ⁡ ,1h ∗ convplP[evi ×at] (Ctr_Op_rampMoins2)

4 Storage constraints

Operation

stoopE t+1 = stoopE t∗(1−stoopα)+ΔT tstoopP tin∗ stoopηin−ΔT tstoopP tout∕ stoopηout[at×evo×d×st] (Ctr_Op_storage_level_definition)
stoopE 0 = stoopE end[at ×evo ×st] (Ctr_Op_storage_initial_level)
stoopPout ≤ stoplPmax ⁡ [at ×evo ×d ×st] (Ctr_Op_storage_power_out_max)
stoopPin ≤ stoplPmax ⁡ [at ×evo ×d ×st] (Ctr_Op_storage_power_in_max)
stoopE ≤ stoplEmax ⁡ [at ×evo ×d ×st] (Ctr_Op_storage_capacity_max)

Planning

stoplC = stoplUC ∗ stoplEmax ⁡ [at ×evo ×st] (Ctr_Pl_planning_storage_capacity_costs)
stoplEmax ⁡ ≥ stoplEmax ⁡ ,min ⁡ [at ×evo ×st] (Ctr_Pl_storage_min_capacity)
stoplEmax ⁡ ≤ stoplEmax ⁡ ,max ⁡ [at ×evo ×st] (Ctr_Pl_storage_max_capacity)
stoplEmax ⁡ = stoplPmax ⁡ ∗ stoopNHmax ⁡ [at ×evo ×st] (Ctr_Pl_storage_max_capacity2)
